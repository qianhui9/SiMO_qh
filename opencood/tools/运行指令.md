# SiMO_qh 两个模块效果运行指令

本文档用于在 `/data/qh/phdCode/work3/SiMO_qh` 中完整评估当前项目的两个新增模块效果：

- `Light-SAD`：运行时 per-CAV 模态调度，控制每个 CAV 使用 LiDAR、Camera 或 LiDAR+Camera。
- `SD-LAMMA`：Supply-Demand-Aware LAMMA Communication，在 LAMMA 后的统一 BEV 特征上做供需感知通信区域选择。

当前可直接运行的权重目录：

- `saved_models/SiMO-PF`：Pyramid Fusion 主路径，包含 `net_epoch27.pth`。两个新增模块都接在这条路径上，建议优先用它看模块效果。
- `saved_models/SiMO-AttFuse`：AttFuse 对照路径，包含 `net_epoch21.pth`，主要用于完整对照评估。

## 8. 保存预测、GT 和可视化

保存预测和 GT 的 `.npy`：

```bash
CUDA_VISIBLE_DEVICES=0 python -u opencood/tools/inference.py \
  --model_dir saved_models/SiMO-PF \
  --fusion_method intermediate \
  --range 51.2,51.2 \
  --save_npy \
  --save_vis_interval 1000
```

想保存更多 BEV 可视化，把 `--save_vis_interval` 调小，例如每 10 帧保存一次：

```bash
CUDA_VISIBLE_DEVICES=0 python -u opencood/tools/inference.py \
  --model_dir saved_models/SiMO-PF \
  --fusion_method intermediate \
  --range 51.2,51.2 \
  --save_vis_interval 10
```

输出位置说明：

- AP 结果会写到对应模型目录下的 `eval_*.yaml`。
- BEV 可视化会写到对应模型目录下的 `vis_*` 文件夹。
- `--save_npy` 会写到对应模型目录下的 `npy/`。
- `--light_sad_dump_state` 会写到 `--light_sad_dump_path` 指定的 JSONL 文件。
- `--sd_lamma_log` 的通信比例、payload、per-modality selected ratio 会直接打印在终端日志中。

## 9. 单模态对照

当前两个模型的配置默认是 LiDAR+Camera 多模态推理：

```yaml
lamma:
  single_mode: false
```

如果要跑单模态，需要修改对应 `saved_models/*/config.yaml` 中的：

```yaml
lamma:
  single_mode: lidar
```

或：

```yaml
lamma:
  single_mode: camera
```

然后重新运行同一条 `inference.py` 命令。跑完单模态后，建议再改回：

```yaml
lamma:
  single_mode: false
```
